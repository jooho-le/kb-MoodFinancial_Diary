## 각 스프린트 별 Daily Standup 가이드

### Daily Standup 구조 (15분)
1. **어제 완료한 작업** (각자 2-3분)
2. **오늘 계획 작업** (각자 1-2분)
3. **블로커/도움 필요사항** (각자 1분)
4. **협업 필요 사항 논의** (3-5분)

### 협업 체크포인트
- **API 인터페이스 변경**: 상대방 작업에 영향 즉시 공유
- **공통 데이터 모델**: 함께 설계 및 리뷰
- **기술적 의사결정**: 페어 프로그래밍 또는 함께 논의

### 일간 기술 리뷰
**매일 오후 (1시간)**:
- 코드 리뷰 및 아키텍처 점검
- 성능 이슈 및 개선 방안 논의
- 다음 주 기술적 챌린지 사전 논의
